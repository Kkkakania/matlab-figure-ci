## 3. 详细检查

### 3.1 尺寸和导出物

- 目标单栏/双栏宽度：
- 当前图件宽高：
- PNG/TIFF 分辨率：
- PDF/SVG/EPS 放大后是否清晰：
- 插入 Word/PPT/LaTeX 后是否二次变糊：
- 修改建议：

### 3.2 字体、线条和标记

- 字体是否统一：
- 字号是否在目标载体中可读：
- 线宽是否足够：
- marker 是否过小或过密：
- 误差棒/置信区间是否清楚：
- 修改建议：

### 3.3 坐标轴、单位和标注

- 坐标轴名称：
- 单位：
- 刻度范围：
- 对数轴、百分比或归一化是否已说明：
- 多 panel 顺序是否一致：
- 修改建议：

### 3.4 配色和可访问性

- 彩色阅读是否容易区分：
- 灰度打印是否仍可区分：
- 红绿组合是否是唯一信息表达：
- 热图是否有 colorbar 和单位：
- 背景是否干扰数据阅读：
- 修改建议：

### 3.5 隐私、来源和可复现

- 输出图是否包含本地绝对路径、用户名或内部项目名：
- 图件是否由脚本生成：
- 脚本和输出文件版本是否对应：
- 数据是否为合成、公开或已脱敏数据：
- 是否存在 `.fig`、`.mat`、`.p`、`.opju` 等难审计素材：
- 修改建议：

## 5. 建议的 `mfigci` 检查

```bash
mfigci scan --config mfigci.yml
mfigci gallery --config mfigci.yml
mfigci check --config mfigci.yml --report mfigci-report.md
```

如果这是 release 前检查，可额外运行：

```bash
mfigci release-preflight --output release-preflight.json
```

## 6. 结论

结论选一项：

- 可提交：基础呈现和发布卫生风险较低，仍需按目标要求复核。
- 建议修改后提交：存在字体、尺寸、导出、标注或 provenance 风险。
- 需要作者确认：图件呈现问题可能影响读者理解，需要作者自行判断。
